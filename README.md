# My Bookshelf

**PDF·TXT 문서를 Obsidian 위키 노트로 바꾸는 개인 연구 도구** — 텍스트 변환 → 챕터 분할 → (영문서) 한국어 번역 → 문서 요약 → Obsidian Wiki 반영을 한 흐름으로 처리합니다.

> 🇬🇧 English manual: [README.en.md](README.en.md)

이 저장소는 macOS와 Windows 양쪽에서 같은 코어(`core/`)를 사용합니다. 설치 방법만 플랫폼별로 다릅니다.

---

## 1. 이 프로그램은 무엇인가

책이나 논문의 PDF·TXT를 넣으면, 다음 단계를 거쳐 **읽기 좋은 요약 위키 노트**를 만들어 Obsidian 보관함(Vault)에 저장합니다.

```
PDF/TXT  →  텍스트 변환  →  챕터 분할  →  영문번역(영문서만)  →  문서요약  →  Obsidian Wiki
```

- 스캔본이 아닌 **텍스트 레이어가 있는 PDF**(또는 TXT)를 대상으로 합니다.
- 번역·요약·위키 생성에는 **AI**를 씁니다. API 키를 넣거나, Claude/ChatGPT 구독 CLI를 켜서 사용합니다.
- 결과 노트는 **저자·핵심 요약·장별 개요·핵심 인용·핵심 키워드**를 담고, 어려운 원전은 용어 원어 병기·풀어 쓰기로 읽기 쉽게 만듭니다.

## 2. 설치

### macOS

1. [최신 릴리스](https://github.com/Brightinyou/my-bookshelf-for-mac/releases/latest)에서 `MyBookshelf-vX.Y.Z-mac.zip`을 받아 압축을 풉니다.
2. `MyBookshelf.app`을 **응용 프로그램** 폴더로 옮깁니다.
3. 처음 실행은 **우클릭 → 열기**를 선택합니다(서명 없는 개인 배포라 뜨는 정상 경고).
4. 첫 실행 시 필요한 파이썬 패키지를 자동 설치합니다(수 분 소요). 이후에는 아이콘을 더블클릭하면 네이티브 창으로 열립니다.

> Python 3.10 이상이 필요합니다. 없으면 [python.org](https://www.python.org/downloads/)에서 설치하세요.

### Windows

1. [최신 릴리스](https://github.com/Brightinyou/my-bookshelf-for-pc/releases/latest)에서 설치 파일을 받습니다.
   - **Chrome**: `Setup.exe`를 그대로 내려받으면 됩니다.
   - **Edge · Internet Explorer**: `.exe` 직접 다운로드가 차단될 수 있습니다. 이때는 같은 릴리스의 **`MyBookshelf-Setup.zip`**을 받아 **압축을 푼 뒤** 안의 `Setup.exe`를 실행하세요. (또는 Edge 다운로드 알림의 `⋯` → **유지(Keep)** → **계속 유지**를 눌러 차단을 풀어도 됩니다.)
2. **⚠️ "알 수 없는 게시자" 경고 안내** — 이 앱은 **코드 서명 인증서가 없는** 개인 배포라, 실행 시 Windows SmartScreen이 파란 화면으로 **"Windows의 PC 보호 — 알 수 없는 게시자"** 경고를 띄울 수 있습니다. 바이러스가 아니라 서명 비용을 들이지 않았기 때문입니다.
   → 파란 창에서 **추가 정보**를 클릭한 뒤 나타나는 **실행** 버튼을 누르면 설치가 진행됩니다.
3. **설치 언어(한국어/English)**를 고릅니다 — 앱 기본 언어가 됩니다.
4. 파이썬 패키지 다운로드로 수 분 걸립니다. 끝나면 자동 실행되고, 이후에는 바탕화면/시작 메뉴의 **My Bookshelf** 아이콘으로 엽니다.

> Python 3.10 이상이 필요합니다. 없으면 `Setup.exe`가 자동 설치를 제안합니다. PDF 텍스트 추출기(Poppler)는 설치본에 포함되어 별도 설치가 필요 없습니다.

## 3. 첫 설정 — AI 연결

앱의 `⚙️ 설정` 탭에서 **둘 중 하나**를 준비합니다. **AI 모델은 설정에서 한 번만 고르면** 모든 단계가 그 모델을 씁니다.

- **AI 구독(CLI)** — API 키 없이 구독으로 사용 (권장). 우선순위가 API 키보다 높습니다.
  - **Claude**: Claude(Pro/Max) 구독자. `claude` CLI 설치·로그인 후 토글을 켜세요.
  - **Codex**: ChatGPT(Plus/Pro) 구독자. `codex` CLI 설치·로그인 후 토글을 켜세요.
- **AI API 키** — Gemini / OpenAI / Anthropic 키를 직접 입력.

설치·로그인 후 앱을 재시작하면 설정 탭에 인식됩니다. 로그인은 브라우저 인증이라 한 번만 하면 됩니다.

또한 `⚙️ 설정 → 옵시디언(Obsidian) 보관함 설정`에서 위키 노트를 저장할 폴더(Vault)를 확인·변경합니다.

## 4. 작업 흐름 상세

상단 메뉴에서 단계를 오갑니다. 각 탭의 파일 업로드 영역은 **파일 선택 또는 끌어다 놓기(Drag & Drop)** 모두 됩니다. 폴더 열기 버튼은 접이식으로 작게 두어 실제 작업 공간이 눈에 띄도록 했습니다.

### ① 📄 텍스트 변환
- PDF/TXT를 올리면 **처리 대기**에 쌓입니다.
- 항목을 골라 **[텍스트 변환 처리]** — 텍스트 레이어를 추출해 TXT로 저장합니다. 원본 PDF는 보관됩니다.
- **[삭제]** 로 잘못 올린 파일을 뺄 수 있습니다.
- **논문 출처로 가져오기**: URL·DOI·arXiv 번호로 논문을 바로 받아올 수 있습니다(로그인·유료 페이지는 직접 PDF를 내려받아 업로드하세요).

### ② 📂 챕터 분할
- 책 TXT를 **챕터 단위 파일**로 나눠 책별 폴더에 저장합니다.
- **[분할 처리]** — 챕터로 분할. **[다음단계로 이동]** — 나눌 필요가 없으면 통째로 다음 단계로 보냅니다(영문→영문번역, 한글→문서요약).
- 짧은 문서는 별도 '짧은 문서 확인'에서 분할/이동을 고릅니다.

### ③ 🌐 영문번역
- 챕터를 한국어로 번역해 `_ko.txt`로 저장합니다.
- 업로드한 TXT도 **번역 대기**에 등록되며, **[▶ 시작]** 을 눌러야 번역이 시작됩니다.
- **영어 UI에서는 이 단계가 숨겨집니다** — 영어 사용자는 영문을 다시 영문으로 번역할 일이 없기 때문입니다(§6 참고).

### ④ 📝 문서요약
- 챕터별 요약 노트(`_wiki.md`)를 만듭니다 — 저자·핵심 요약·개요·핵심 인용·핵심 키워드(해설 포함).
- 업로드·대기 항목을 골라 **[▶ 시작]**.

### ⑤ 📖 위키반영
- 요약을 합쳐 Obsidian 보관함에 **허브 노트 + 장별 노트**로 저장합니다.
- 대기 목록에서 **[전체 선택]/[해제]** 후 **[▶ 시작]**.

---

## 5. 시작 · 중단 · 이어하기

챕터 분할·영문번역·문서요약·위키반영의 AI 작업은 **[▶ 시작]** 을 누르면 처리 화면만 남고 다른 기능·탭 이동이 **잠깁니다**(실수로 작업을 벗어나는 것 방지).

- 처리 화면에는 진행률과 항목별 결과가 표시됩니다.
- **[■ 중단]** 을 누르면 **현재 항목까지 마친 뒤** 멈추고 전체 화면이 돌아옵니다.
- 남은 작업은 대기 목록에 그대로 남아 **[▶ 시작]** 을 다시 누르면 이어서 처리됩니다.
- 대기 목록에서 **[🗑 삭제]** 로 잘못 넣은 작업을 뺄 수 있습니다.

## 6. 언어 설정과 번역 단계

`⚙️ 설정 → 언어`에서 한국어/English를 바꿀 수 있습니다.

- **English로 바꾸면** 영문번역 단계가 메뉴·내비게이션·파이프라인에서 **사라집니다**. 영어 문서를 다시 영어로 번역할 필요가 없기 때문이며, 챕터 분할 후 문서는 곧바로 요약 단계로 이동합니다.
- 한국어로 되돌리면 영문번역 단계가 다시 나타납니다.

## 7. 데이터 위치

기본 데이터 폴더(설치 언어에 따라 폴더명이 한글/영문):

```
0_업로드대기/     업로드·다운로드 대기 (처리 전)
1_원본PDF/        원본 PDF 보관
2_변환TXT/        변환된 TXT (완료/ = 분할 끝난 원본 보관)
3_챕터/<책>/       챕터·번역(_ko)·요약(_wiki.md)·전체요약이 함께 있는 작업장
실패/, 로그/       실패 파일·로그
```

위키 노트는 별도 Obsidian 보관함(Vault)에 저장됩니다(`⚙️ 설정`에서 선택). 설정은 `~/.config/mybookshelf/config.json`(macOS/Linux) 또는 사용자 프로필의 같은 경로에 저장됩니다.

## 8. 문제 해결

- **"사용 가능한 AI가 없습니다"** — `⚙️ 설정`에서 API 키를 넣거나 CLI 구독(Claude/Codex)을 켜세요.
- **업데이트했는데 옛 화면이 보임** — 앱을 완전히 종료 후 다시 여세요(실행 중인 서버가 남아 있을 수 있습니다).
- **스캔 PDF** — 텍스트 레이어가 없는 스캔본은 먼저 OCR로 텍스트 PDF/TXT로 만든 뒤 넣으세요.
- (Windows) 설치·실행 오류는 설치 폴더의 `install.log` / `launch-error.log`를 확인하세요.

## 개발자용

```
core/                앱 핵심 코드
  pipeline_app.py    Streamlit UI (전 단계)
  services/          처리 로직 (convert/translate/chapters/wiki/i18n …)
  chapter_wiki.py    챕터 분할 + 요약 생성 (멀티 공급자 AI)
  llm_providers.py   AI 공급자 추상화 (Gemini/OpenAI/Anthropic/Claude CLI/Codex CLI)
  .streamlit/        config.toml (라이트 테마·개발자 툴바 비활성)
dev/                 빌드 스크립트 (build_mac_app.sh, bump_version.py …)
```

- macOS 빌드: `dev/build_mac_app.sh` → `dist/mac/MyBookshelf.app`
- 개발 실행(레포 코드): 각 플랫폼의 `start` 스크립트 또는 `streamlit run core/pipeline_app.py`
