## 2026-06-25 — 네이티브 창 래핑 (PyWebView)

| 파일 | 변경 내용 |
|------|-----------|
| `core/desktop.py` (신규) | Streamlit을 헤드리스 백그라운드로 기동 → 포트 준비 대기 → 주소창 없는 PyWebView 창. 창 닫으면 서버 종료(좀비 방지), 기존 서버 재사용 |
| `requirements.txt` | `pywebview` + `pyobjc-framework-WebKit`(mac) / `pythonnet`(win) |
| `start.command`/`start.bat` | `desktop.py` 실행, `import webview` 실패 시 브라우저 폴백 |
| `start-app.vbs` | `pythonw`로 콘솔 없이 `desktop.py` 실행 |

- **엔진**: macOS=WebKit(내장), Windows=Edge WebView2(Win11 내장, Win10 구버전은 부트스트래퍼 필요)
- **검증**: macOS 네이티브 창 시각 확인 ✓. 클린 복사본에서 setup→start 전 흐름 통과(venv 1.5G, pywebview+docling+torch, 콜드스타트 서버 생명주기·좀비 없음) ✓. **Windows WebView2 실기 미검증**(머신 없음)
